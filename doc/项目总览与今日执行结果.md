# 项目总览与今日执行结果

更新时间：2026-05-27

## 1. 项目一句话

Peak Ball 是一个足球比赛数据与预测分析项目。当前由前台 H5、管理后台、前台 API、管理 API、独立爬虫和 MySQL 数据库组成。独立爬虫抓取当天比赛、历史战绩、欧赔和盘口数据，写入 MySQL；前台 API 读取这些数据；前台 H5 展示今日比赛列表、比赛详情和数据结论。

## 2. 模块和端口

| 模块 | 目录 | 默认端口 | 作用 |
|---|---|---:|---|
| 前台 H5 | `src/` | 5173 | 用户端比赛列表、详情、分析、个人页等 |
| 管理后台 | `admin/src/` | 5174 | 后台数据查看、爬虫任务、系统管理 |
| 前台 API | `go_server/` | 18080 | 前台页面读取比赛、赔率、历史、用户等接口 |
| 管理 API | `go_admin/` | 8081 | 管理后台登录、权限、爬虫数据管理接口 |
| 独立爬虫 | `go_crawler/` | 无 | 抓取今日比赛、历史、欧赔、盘口并入库 |
| 数据库 | MySQL | 3306 | 默认库名 `football_data` |

## 3. 当前数据链路

1. `go_crawler` 请求 `https://vipc.cn/i/live/football/date/today/next` 获取今日比赛列表。
2. 对每场比赛继续请求：
   - `https://vipc.cn/i/match/football/{matchId}/history`
   - `https://vipc.cn/i/match/football/{matchId}/odds/euro`
   - `https://vipc.cn/i/match/football/{matchId}/odds/pankou`
3. 数据写入 MySQL：
   - `moneys`：比赛基本信息
   - `history_moneys`：交锋、近况、排名、未来赛程 JSON
   - `odds_moneys`：欧赔数据 JSON
   - `pankou_moneys`：亚盘和大小球 JSON
4. `go_server` 提供前台接口：
   - `GET /api/matches?date=YYYY-MM-DD`
   - `GET /api/match/:id`
   - `GET /api/match/:id/history`
   - `GET /api/match/:id/odds/euro`
   - `GET /api/match/:id/odds/pankou`
5. 前台 H5 通过 Vite 代理 `/api` 到 `http://localhost:18080`，页面直接展示今日比赛和详情分析。

## 4. 启动顺序

```powershell
# 1. 前台 API
cd go_server
go run .

# 2. 管理 API
cd ..\go_admin
go run .

# 3. 前台 H5
cd ..
npm install
npm run dev

# 4. 管理后台
cd admin
npm install
npm run dev
```

首次需要管理后台账号时执行：

```powershell
mysql -uroot -p123456 football_data < go_admin/seed.sql
```

## 5. 已实现功能

### 前台 H5

- 首页读取今日比赛列表。
- 首页按联赛筛选比赛。
- 首页展示球队名称、Logo、比分、比赛时间和状态。
- 比赛详情页读取基本信息、历史战绩、欧赔和盘口。
- 比赛详情页展示胜平负隐含概率、近期状态、历史交锋和今日结论。
- 欧赔缺少源站平均值时，前端会从全部公司赔率计算平均欧赔。

### 前台 API

- 比赛列表、比赛详情、历史战绩、欧赔、盘口接口已实现。
- 静态图片目录 `/footballimg` 已映射到 `public/footballimg`。
- 启动时通过 GORM AutoMigrate 自动建表/更新表。

### 独立爬虫

- 拉取当天比赛列表。
- 逐场拉取历史战绩、欧赔、盘口。
- 下载球队 Logo 到 `public/footballimg`。
- 使用随机请求头和 1.5 秒间隔控制请求频率。
- 数据冲突时按 `match_id` 更新已有记录。
- 欧赔缺少源站平均值时，爬虫会从全部公司赔率计算平均欧赔并写入 `avg_odds`。

### 管理后台和管理 API

- 管理后台登录、用户、角色、菜单、权限、操作日志页面已接入。
- 爬虫数据列表、详情、删除、同步任务、任务日志页面已接入。

## 8. 今日分析口径

- 欧赔：优先使用 `avg_odds`；如果为空，从 `odds_moneys.data` 的全部公司赔率计算平均值。
- 概率：按胜/平/负平均赔率倒数归一化得到隐含概率。
- 近况：用近期汇总的 `win * 3 + draw` 对比主客队状态。
- 交锋：用 `againstSummary.win` 与 `againstSummary.lose` 判断历史交锋方向。
- 信心：最高隐含概率 >= 50% 为高信心，>= 42% 为中信心，其余为谨慎观察。
