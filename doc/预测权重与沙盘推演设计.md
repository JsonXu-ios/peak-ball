# /analysis 庄家盈亏测算

## 当前功能

- `/analysis` 只保留庄家盈亏测算，不再展示受众、综合评分、用户 ROI。
- 测算基数固定为 `50000000` 元。
- 投注分布优先使用竞彩官方 `jczq/lr/{matchId}` 接口 `tzbl` 的主胜、平局、客胜支持率；缺失时回退散户心理数据。
- 赔率来源包含竞彩、威廉 `companyId=115`、Bet365 `companyId=281` 三组胜平负赔率。
- 竞彩胜平负赔率和竞彩投注比例来自 `https://www.vipc.cn/i/match/jczq/lr/{matchId}` 的 `tzbl`，并随欧赔任务一起拉取到 `odds_moneys.sporttery_trade`。
- 竞彩交易盈亏来自 `jyykSpf`，让球交易盈亏来自 `jyykRqspf`。
- 每个赔率来源按主胜、平局、客胜三个最终赛果单独结算庄家赔付和盈亏。

## 已移除功能

- 5 类受众模型。
- 10000 人、人均 1000 元的用户沙盘。
- 稳健、激进、反热门、盘口、数据等受众选择。
- 综合评分和评分拆解。
- 用户侧全场 ROI、命中受众、受众收益推演。

## 计算字段

- `totalStake`：总投注额，固定 `50000000`。
- `retailDistribution`：投注比例，包含主胜、平局、客胜百分比。
- `markets`：赔率来源列表，包含竞彩、威廉、Bet365。
- `odds`：某个赔率来源的主胜、平局、客胜赔率。
- `betStake`：某方向投注金额。
- `payout`：该方向打出时需要返还给买中散户的金额。
- `bookmakerProfit`：本地按 5000 万总池测算的庄家净盈亏金额，按 `totalStake - payout` 计算。
- `bookmakerLoss`：本地按 5000 万总池测算的庄家净亏损金额，按 `payout - totalStake` 计算；正数表示庄家亏损，负数表示庄家盈利。
- `bookmakerRoi`：本地净盈亏金额占总投注额的比例。
- `officialProfitRate`：竞彩交易盈亏接口中的庄家盈亏率，直接读取 `hy/dy/ay`，不使用本地 5000 万测算兜底。
- `probability`：竞彩投注比例接口中的赛果概率，仅竞彩来源返回。
- `error`：竞彩投注比例接口中的支持率误差，仅竞彩来源返回。
- `hotColdIndex`：竞彩交易盈亏接口中的冷热指数，仅竞彩来源返回。

## 公式

```text
totalStake = 50000000
retailShare[result] = sportterySupportRate[result] / sum(sportterySupportRate)
betStake[result] = totalStake * retailShare[result]
payout[market][result] = betStake[result] * odds[market][result]
bookmakerProfit[market][result] = totalStake - payout[market][result]
bookmakerLoss[market][result] = payout[market][result] - totalStake
bookmakerRoi[market][result] = bookmakerProfit[market][result] / totalStake * 100
```

竞彩投注比例缺失时使用散户心理数据：

```text
retailShare[result] = sanhuxinli[result] / sum(sanhuxinli)
```

散户心理（sanhuxinli）不再是纯欧赔隐含概率，而是加权模拟分布（纯本地计算，不使用竞彩数据）：

```text
base = 平均欧赔隐含概率（归一化）
sideShift（主队方向偏移，百分点，clamp ±12）:
  + clamp(亚盘即时盘 * 3, ±6)           # 散户追深盘强队
  + clamp((即时盘 - 初盘) * 8, ±4)       # 升盘追强、退盘流出
  + clamp((历史胜率 - 历史负率) * 0.06, ±4)  # 交锋占优跟风（样本不足剔除）
  + clamp(近5场净胜球差 * 3, ±4)         # 最近状态（样本不足剔除）
  + clamp((期望让球均值 - 即时盘) * 4, ±3)   # 均值比盘口深→强方被低估
drawShift（平局偏移，clamp ±5）:
  clamp((2.5 - goalAnchor) * 6, ±5)，goalAnchor = 大小球即时盘*0.6 + 球数均值*0.4
home/away ± sideShift；draw + drawShift（差额按主客占比分摊）；
各项 clamp [5, 85] 后做羊群放大，再归一化到 100%：
  p'ᵢ = pᵢ^γ / sum(pⱼ^γ)，γ = retailHerdingGamma = 1.8
  # 模拟散户对热门的非线性集中；均势场几乎不受影响；γ=1 即关闭。
```

注意：加权散户心理只影响 sanhuxinli 展示、本地测算（localMarket）与模拟盘的
投注分布端。推荐、两队历史、球数统计、让球压力、大小球等基础数据的统计与
计算方式均不经过该函数，保持原口径不变。

## 示例

```text
总投注额 = 5000万
主胜竞彩支持率 = 30%
主胜投注额 = 5000万 * 30% = 1500万
主胜威廉赔率 = 1.2
主胜打出返还 = 1500万 * 1.2 = 1800万
主胜打出威廉庄家盈亏 = 5000万 - 1800万 = 3200万
主胜打出威廉庄家 ROI = 3200万 / 5000万 * 100 = 64%
```

## 页面展示

- `竞彩投注比例`：按截图表格展示赛果、指数、概率、彩民支持率、误差、心理误差。
- `竞彩交易盈亏`：展示赛果、指数、投注比例、交易额、庄家赔付、本地净盈亏、庄家盈亏、冷热指数。
- `竞彩让球交易盈亏`：展示让球胜平负的赛果、指数、投注比例、交易额、庄家赔付、本地净盈亏、庄家盈亏。
- `威廉交易盈亏`：使用竞彩投注比例和威廉胜平负赔率计算庄家赔付与盈亏。
- `Bet365交易盈亏`：使用竞彩投注比例和 Bet365 胜平负赔率计算庄家赔付与盈亏。
