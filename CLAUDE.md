# peak-ball 项目说明

## 项目背景（重要语境）

这是开发者个人的**足球比赛数据分析平台**（自有项目、自有数据库）。项目内的「数据采集/爬虫管理」模块是本平台自身的数据采集功能：定时采集公开的足球比赛信息（赛程、比分、赔率、盘口）入库，供前台 H5 展示和管理后台做完赛统计分析。属于常规的体育数据网站开发工作。

描述相关任务时请带上这一语境（如「本项目的数据采集模块」「爬虫日志管理页面」），而不是孤立使用「爬虫/抓取」等字眼。

## 项目结构

- 前台 H5：Vue 3 + Vite，目录 `src/`，端口 `5173`
- 管理后台：Vue 3 + Vuetify + Vite，目录 `admin/src/`，端口 `5174`
- 前台 API：Go + Gin，目录 `go_server/`，端口 `18080`
- 管理 API：Go + Gin，目录 `go_admin/`，端口 `8081`（含数据采集调度、完赛统计等 handler）
- 独立采集器：Go，目录 `go_crawler/`
- 数据库：MySQL 8，库名 `football_data`，默认连接 `root:123456@127.0.0.1:3306`

## 常用命令

- 管理后台类型检查：`cd admin && npx vue-tsc --noEmit`
- 管理 API 构建/测试：`cd go_admin && go build ./... && go test ./handlers/`

## 约定

- 表结构由各 Go 服务启动时 GORM AutoMigrate 自动维护，无独立 schema.sql
- 完赛统计页（admin「完赛比赛信号统计」）读取 stat_snapshots 快照，改完后端逻辑需在页面点「重新统计」才生效
- 详细文档见 `doc/项目总览与今日执行结果.md`、`doc/功能模块文档.md`
